## 一、VPN 的基本原理

想象你住在一个小区里，小区门卫（防火长城/GFW）会检查每一件快递，看收件地址是不是"违禁目的地"。如果是，就直接扣押。

**普通上网**的过程是这样的：

```
你的电脑  ──[明信片，地址一览无余]──▶  门卫检查  ──✗──▶  Google（被拦截）
```

**代理/VPN 上网**的过程变成了这样：

```
你的电脑  ──[密封包裹，写的是"快递中转站"的地址]──▶  门卫放行
                                                           │
                                        抵达境外中转站（节点服务器）
                                                           │
                                          解包，真正转发至 Google
                                                           │
                                          结果原路返回给你
```

这里有几个关键动作：

|动作|技术术语|作用|
|---|---|---|
|把数据装进"密封包裹"|**加密（Encryption）**|让门卫看不懂内容|
|把目的地改成中转站|**隧道（Tunnel）**|让门卫以为你在访问合法站点|
|中转站替你访问目标|**代理（Proxy）**|真正的请求由境外服务器发出|

### 核心概念：本质是"借道"

VPN（Virtual Private Network）和代理（Proxy）本质上都是**借助第三方服务器中转流量**，区别在于：

- **VPN**：在操作系统层面建立加密隧道，**所有流量**强制走隧道，就像在你电脑和服务器之间挖了一条专用地道
- **代理（Proxy）**：更灵活，可以只让部分流量走代理，其余流量照常直连

现在主流的"科学上网"工具（如 Clash）本质上是**代理客户端**，而非传统意义上的 VPN，但大家约定俗成地混用这两个词。

## 二、[软件 vs 节点：工具与通道的关系](https://login.doumao.vip/#/register?code=40P3NM6w)

这是很多人最容易搞混的地方。用一个比喻来说：

> **软件（如 Clash）= 汽车** **节点（如机场提供的服务器）= 道路/高速公路**

你有一辆性能卓越的跑车（Clash），但如果没有路（节点），它哪儿也去不了。反过来，有路没车，你也出不了门。

### 软件做的事

代理软件负责：

- 在本地监听你电脑发出的网络请求
- 按照规则判断哪些流量要走代理、哪些直连
- 将需要代理的流量按照协议（VMess、Trojan、Shadowsocks 等）**打包加密**，发送到节点服务器
- 接收节点返回的数据并解密，交还给你的应用程序

```
应用程序（Chrome、YouTube）
       │
       ▼
  Clash（本地代理客户端）
  ┌─────────────────────┐
  │  • 监听本地端口      │
  │  • 规则匹配          │
  │  • 协议加密/封装     │
  └─────────────────────┘
       │
       ▼（加密流量）
  节点服务器（机场）
       │
       ▼（解密，以服务器IP访问）
  目标网站（Google、YouTube）
```

### 节点（机场）做的事

"机场"是对**代理节点服务提供商**的俗称（因为节点名字里常见各种城市/国家，像机场目的地）。机场提供的是：

- **境外服务器资源**：在香港、日本、美国等地部署服务器
- **代理协议支持**：服务器端运行 VMess、Trojan、VLESS、Shadowsocks 等协议
- **订阅链接**：一个 URL，里面包含了所有节点的地址、端口、密码等信息

你把订阅链接填入 Clash，Clash 就知道"我有哪些节点可以用"了。

### 常见代理协议简介

|协议|特点|
|---|---|
|**Shadowsocks (SS)**|老牌协议，轻量，但特征较明显|
|**VMess**|V2Ray 开发，伪装成 HTTPS 流量，抗封锁强|
|**Trojan**|直接伪装成 HTTPS，极难被识别|
|**VLESS**|VMess 的简化版，更高效|
|**Hysteria2**|基于 QUIC/UDP，速度极快，适合高延迟网络|

## 三、什么是"内置节点"的软件

市面上有一类软件，把软件和节点**打包在一起**销售，用户无需自己找机场，开箱即用。典型的有：

- **Lantern（蓝灯）**：内置自己的服务器节点，无需配置
- **老王 VPN、快连 VPN** 等：同样内置节点

### 内置节点软件的优缺点

|对比项|内置节点软件|Clash + 独立机场|
|---|---|---|
|上手难度|⭐ 极低，点开即用|⭐⭐⭐ 需要配置|
|灵活性|❌ 无法自定义|✅ 高度可定制|
|节点质量|参差不齐，通常较差|取决于机场质量|
|安全性|⚠️ 存疑，数据过第三方|相对透明，可审计|
|价格|常有免费版（限速/广告）|机场通常按月付费|
|速度/稳定性|一般|好机场可以非常稳定|

**安全提示**：免费内置节点的软件，其盈利模式往往是收集用户流量数据或展示广告。如果对隐私有要求，**建议使用开源的 Clash 配合可信机场**。

## 四、Clash 各项设置详解

下面从网络技术角度，逐一解释 Clash（及其图形化版本 Clash Verge、Clash for Windows 等）中的各类设置。

---

### 4.1 系统代理（System Proxy）

#### 它是什么？

**系统代理**是操作系统提供的一个统一代理入口。当你在 Clash 里开启"系统代理"时，Clash 会向操作系统注册一条规则：

> "所有支持系统代理的应用，请把你的 HTTP/HTTPS 流量发给我（本地的 127.0.0.1:7890）。"

```
Chrome、Edge 等浏览器
        │（读取系统代理设置）
        ▼
  127.0.0.1:7890（Clash 监听的本地端口）
        │
        ▼
  Clash 处理并转发
```

#### 局限性

系统代理只对**主动遵守系统代理设置的应用**有效。大多数浏览器会遵守，但：

- 很多游戏客户端、命令行工具（curl、wget）**默认不走系统代理**
- UWP 应用（Windows 商店应用）由于沙盒机制，**无法使用系统代理**
- 部分 App 有硬编码的直连逻辑，完全绕过系统代理

这就是为什么需要 TUN 模式。

---

### 4.2 TUN 模式

#### 它是什么？

TUN 是 Linux/macOS/Windows 内核提供的一种**虚拟网络设备**（TUN/TAP）。Clash 的 TUN 模式会：

1. 创建一块**虚拟网卡**（如 `utun0` 或 `clash-tun`）
2. 在路由表中添加规则，把**所有流量**（或指定流量）路由到这块虚拟网卡
3. Clash 接管这块虚拟网卡，从而**捕获到所有网络流量**，无论应用是否遵守系统代理

```
所有应用程序
     │（操作系统网络栈）
     ▼
  路由表：默认路由 → clash-tun 虚拟网卡
     │
     ▼
  Clash TUN 接口（工作在第三层/IP层）
     │
     ▼
  规则匹配 + 代理转发
```

#### 系统代理 vs TUN 模式对比

|特性|系统代理|TUN 模式|
|---|---|---|
|工作层次|应用层（L7）|网络层（L3）|
|覆盖范围|只有遵守代理的应用|几乎全部流量|
|UDP 支持|❌ 基本不支持|✅ 支持（游戏、视频通话受益）|
|需要权限|不需要|需要管理员/root 权限|
|适用场景|日常浏览|游戏、全局代理、命令行工具|

> **💡 实用建议**：日常浏览只开系统代理即可，打游戏或需要完全代理时开 TUN 模式。

### 4.3 规则与分流（Rules / Policy）

#### 核心思想：不是所有流量都要走代理

代理有成本（速度、延迟、流量），访问国内网站走代理反而更慢。**规则系统**就是让 Clash 智能判断：

```
收到请求: google.com
   │
   ▼
规则匹配（从上到下）：
   ├── DOMAIN-SUFFIX,google.com → 走"代理"节点组 ✓ 命中，走代理
   ├── GEOIP,CN → 走"直连"
   └── MATCH → 走"代理"（兜底）
```

#### 常见规则类型

|规则类型|示例|含义|
|---|---|---|
|`DOMAIN`|`DOMAIN,google.com`|完整域名匹配|
|`DOMAIN-SUFFIX`|`DOMAIN-SUFFIX,google.com`|域名后缀匹配（含子域名）|
|`DOMAIN-KEYWORD`|`DOMAIN-KEYWORD,google`|域名关键词匹配|
|`IP-CIDR`|`IP-CIDR,192.168.0.0/16`|IP 段匹配|
|`GEOIP`|`GEOIP,CN`|IP 归属地（国家/地区）|
|`PROCESS-NAME`|`PROCESS-NAME,steam.exe`|按进程名匹配|
|`MATCH`|`MATCH`|兜底规则，匹配所有|

#### 节点组（Proxy Group）

规则最终指向一个**节点组**，而不是具体节点，常见节点组类型：

- **`select`**：手动选择，你点哪个用哪个
- **`url-test`**：自动测速，选最快的节点
- **`fallback`**：自动故障转移，当前节点挂了就换下一个
- **`load-balance`**：负载均衡，多节点轮流使用

### 4.4 自定义转发（端口转发 / 外部控制器）

#### 本地监听端口

Clash 默认在本地开启几个端口：

|端口|用途|
|---|---|
|`7890`|HTTP/HTTPS 代理端口|
|`7891`|SOCKS5 代理端口|
|`7892`|Redir 透明代理端口（Linux）|
|`9090`|外部控制器（RESTful API）|

你可以手动把应用的代理设置指向这些端口，实现"手动指定代理"而不依赖系统代理。

```bash
# 比如在终端里让 curl 走 Clash 代理
curl -x http://127.0.0.1:7890 https://google.com

# 让 git 走代理
git config --global http.proxy http://127.0.0.1:7890
```

#### 允许局域网（Allow LAN）

开启后，Clash 监听地址从 `127.0.0.1`（仅本机）变为 `0.0.0.0`（所有网卡），这样**同一局域网内的其他设备**（手机、平板）可以把代理设置指向你的电脑 IP，共用你的 Clash。

```
手机（192.168.1.5）
  代理设置: 192.168.1.100:7890
       │
       ▼
电脑 Clash（192.168.1.100:7890）
       │
       ▼
  节点服务器
```

### 4.5 DNS 设置

DNS 是 Clash 中最容易被忽视但极其关键的模块。

#### DNS 污染问题

访问 `google.com` 时，你的电脑首先要查询它的 IP 地址。这个 DNS 查询本身也会被拦截和**污染**（返回错误的 IP）：

```
查询: google.com → ?
  ↓（发往运营商 DNS）
运营商 DNS：返回一个假 IP（如 127.0.0.1）
  ↓
你拿着假 IP 去连接 → 失败
```

#### Clash 的 DNS 解决方案

Clash 内置 DNS 模块，可以：

|配置项|作用|
|---|---|
|`fake-ip` 模式|给所有域名分配虚假本地 IP，等实际连接时再通过代理解析真实 IP，彻底防止 DNS 污染|
|`redir-host` 模式|在本地解析域名，再将流量发给代理|
|nameserver|国内 DNS（用于解析国内域名，如 `223.5.5.5`）|
|fallback|国外加密 DNS（用于解析被污染域名，如 `8.8.8.8`、`1.1.1.1`）|
|`enhanced-mode`|开启增强 DNS 处理|

```yaml
# Clash 配置示例（DNS 部分）
dns:
  enable: true
  enhanced-mode: fake-ip
  nameserver:
    - 223.5.5.5     # 阿里 DNS（国内）
    - 114.114.114.114
  fallback:
    - tls://8.8.8.8:853      # Google DNS over TLS
    - https://1.1.1.1/dns-query  # Cloudflare DoH
  fallback-filter:
    geoip: true
    ipcidr:
      - 240.0.0.0/4
```

### 4.6 混合端口（Mixed Port）

较新版本的 Clash 支持 `mixed-port`，即一个端口同时支持 HTTP 和 SOCKS5 协议，使用更方便：

```yaml
mixed-port: 7890   # 代替分开的 port: 7890 和 socks-port: 7891
```

### 4.7 外部控制器与 Secret（API 鉴权）

```yaml
external-controller: 127.0.0.1:9090
secret: "your_password_here"
```

- **外部控制器**：Clash 暴露一个 RESTful API，图形界面（如 Clash Dashboard、Yacd）通过这个接口来控制 Clash、切换节点、查看流量
- **Secret**：API 的访问密码，防止局域网内其他人随意控制你的 Clash

### 4.8 Profile / 订阅管理

**订阅（Subscription）** 本质上是一个 URL，访问后返回一段 YAML 或 Base64 编码的配置文件，里面包含所有节点信息。

```
订阅 URL → Clash 下载解析 → 得到节点列表
```

- **自动更新**：可以设置每隔几小时自动拉取最新订阅（机场节点会变动）
- **覆盖规则（Override）**：在不修改订阅原文的情况下，用本地规则"覆盖"或"追加"配置，适合有自定义需求的用户
